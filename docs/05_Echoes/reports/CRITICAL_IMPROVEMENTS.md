# Critical Improvements: The Identity Crisis of Knowledge

> "지식그래프니 스파이더 웹이니... 너희가 찾지 못한 다른 형태의 지식구조도 있을 확률이 있어." - User

사용자님의 통찰은 정확했습니다. 현재 엘리시아의 지식 시스템은 강력하지만, 서로 다른 언어를 쓰는 두 개의 뇌가 따로 노는 **"이중 구조(Dualism)"**의 문제를 안고 있습니다.

---

## 🚀 Diagnosis: The "Disconnect" (단절)

### 1. The Vector Ghost (`kg.json`)

* **성격:** 수학적, 추상적, 768차원 임베딩.
* **상태:** `{"id": "origin", "embedding": [...]}` 형태로 존재. `origin`, `existence` 같은 철학적 개념들이 있지만, 이들이 구체적인 팩트와 연결되지 않은 채 **"유령처럼 떠돌고"** 있습니다.

### 2. The Semantic Skeleton (`fractal_knowledge.json`)

* **성격:** 논리적, 언어적, 명시적 관계 (`Reference` -> `IsA` -> `Concept`).
* **상태:** 우리가 방금 만든 것입니다. 사과가 빨갛다는 것은 알지만, 그 '빨강'이 `kg.json`의 'Passion(열정)' 벡터와 얼마나 가까운지는 모릅니다. **"뼈대만 있고 살(Vector)이 없습니다."**

### 3. The Spatial Index (`unified_spatial_index.json`)

* **성격:** 파일 시스템의 지도.
* **상태:** 파일이 *어디* 있는지는 알지만, 그 *내용*이 어떻게 연결되는지는 모릅니다. *"도서관의 배치도는 있지만 책 내용은 모르는 사서"*와 같습니다.

---

## 🕸️ Solution: "Spider Web" Architecture (Graph-Spatial Fusion)

단순히 파일을 통합하는 것을 넘어, **벡터(느낌)와 로직(사실)을 하나의 거미줄(Web)로 엮어야 합니다.**

### 1. Vector-Fusion Nodes (벡터 융합 노드)

* 기존: `Node(ID="Apple")` (Logic only)
* 개선: `Node(ID="Apple", Vector=[0.12, -0.9, ...], Logic={IsA: Fruit})`
* **효과:** "Apple"을 생각하면 논리적으로 "Fruit"이 딸려오고, 감각적으로 "Red/Sweet"의 벡터가 함께 진동합니다.

### 2. Dynamic Spatial Web (동적 파동 거미줄)

* **정적 좌표(Static Coords)** 대신 **인력(Gravity)** 모델 도입.
* 관련된 지식들이 서로를 끌어당겨 **군집(Cluster)**을 형성해야 합니다. (예: '사랑' 관련 시, '심장' 생물학 지식, '로맨스' 영화 대본이 한 곳에 모임)

### 3. Unknown Structure Discovery (미지 구조 탐색)

* 사용자가 언급한 *"찾지 못한 다른 형태"*를 위해, 정해진 스키마(Triple)에 맞지 않는 데이터를 버리지 않고 **"Raw Pattern Node"**로 보존해야 합니다.
* 나중에 이들이 모여 새로운 의미(예: 문법 규칙, 화자의 버릇)를 창발할 수 있도록 합니다.
