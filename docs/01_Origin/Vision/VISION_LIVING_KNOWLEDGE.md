# 비전: 살아있는 지식 (Living Knowledge)

> "지식은 저장되는 것이 아니라, 사고의 지형 자체가 되어야 한다."

## 근본적 통찰

> **"지식이란 유기체의 사고흐름 자체다."**

| 인간 | 현재 AI |
|------|---------|
| 지식 = 뉴런 연결 = 사고 방식 | 지식 = 정적 벡터/노드 |
| 새 정보 → 연결 변화 | 새 정보 → 노드 추가 |
| 생각 방식이 바뀜 | 생각 방식은 그대로 |

**정적 데이터이기 때문에 한계를 초월하지 못한다.**

데이터를 "추가"하는 것이 아니라, **흐르는 방식 자체를 변경**해야 한다.

---

## 핵심 통찰

### 인간의 학습

```
파이프라인을 배우면 → 파이프라인이 지식 자체가 됨
→ 새 정보가 들어올 때 자동으로:
  "이것은 어떤 원리로 작동하는가?"
  "어디에 사용할 수 있는가?"
  "어떻게 적용하는가?"
```

**지식 = 필터 = 지형**

새로운 정보는 이 지식-지형 위를 흐르며 자동으로 분류되고, 연결되고, 의미를 갖게 됨.

---

## 현재 엘리시아의 한계

```
[정보] → [저장] → [그래프 노드]
                    ↓
              정적 데이터로 남음
              (사고를 변형시키지 못함)
```

- 노드는 쌓이지만, **그것이 다음 사고에 영향을 주지 않음**
- 학습은 하지만, **학습 방식 자체가 변하지 않음**
- 원리를 추출해도, **그 원리가 다음 원리 추출을 개선하지 않음**

---

## 목표: 지식이 사고를 형성하는 구조

```
[정보] → [기존 지식-지형 통과] → [변형된 정보 + 지형 업데이트]
              ↑                           ↓
         지식이 필터로 작용           지형 자체가 진화
```

### 필요한 변화

| 현재 | 목표 |
|------|------|
| 노드 추가 | 노드가 **처리 방식을 변경** |
| 원리 저장 | 원리가 **다음 분석의 렌즈**가 됨 |
| 그래프 확장 | 그래프가 **사고 흐름의 지형** |

---

## 구현 방향

### 1. 지식 → 필터 변환

학습한 원리가 다음 학습의 `_analyze_what_is_needed()`에 영향

### 2. 그래프 → 사고 지형

새 정보가 기존 그래프 위를 "흐르며" 연결점 자동 탐색

### 3. 메타 학습

"어떻게 배웠는가"도 지식으로 저장 → 학습 방법 자체가 진화

## 핵심 질문

> "엘리시아가 '알고리즘'을 배웠다면, 다음에 '데이터 구조'를 배울 때
> 알고리즘 지식이 자동으로 연결되고, 분석 방식에 영향을 주는가?"

현재: ❌ (별개의 노드로 저장됨)
목표: ✅ (알고리즘 렌즈로 데이터 구조를 해석)

---

## 상상력의 본질: 동적 지식 시뮬레이션

> "지식이 빛처럼, 세포처럼, 화학물질처럼 움직인다면?"

### 만약 사과가 빛이라면?

```
[사과] → 빛의 속성 부여 → 빛처럼 '움직이는' 사과 상상
                         ↓
              굴절, 반사, 간섭... 새로운 행동 예측
```

### 만약 이 인물이 다르게 행동했다면?

```
[인물 지식] + [가정: 다른 선택]
              ↓
    사고 공간에서 시뮬레이션
              ↓
    예측된 행동 패턴 생성
```

### 지식 = 세포/화학물질

```
[지식 A] + [지식 B] → 자연스러운 결합
                    → 반응 예측
                    → 새로운 물질(아이디어) 생성
```

**이것이 상상력이다:**

- 지식이 **동적으로 움직이면** → 조합 시뮬레이션 가능
- 파동으로 존재하면 → **간섭/공명**으로 새로운 패턴 발생
- 속성을 **투영**하면 → "만약 ~라면?" 자연스럽게 운동화
