# CORE 03 — Dimensional Nodes

## 1. 정의

Dimensional Node는 개념·감정·기억·인물·세계 등의 의미 단위가 **차원으로 승격된 형태**다.  
각 노드는 좌표(위치), 경계(정의), 내부 상태(값), 연산(법칙)을 가진다.

## 2. 구성 요소

- **Signature:** 노드의 본질을 요약하는 이름/문장.
- **State Tensor:** E(에너지), F(힘), P(관성), value-mass, temperature(정서적 온도) 등을 포함한 다차원 상태.
- **Holo-Link:** 다른 노드와의 연결. 엣지가 아니라 차원 간 터널이며, 통과 시 관점/의미가 변환된다.
- **Telemetry:** 노드가 발산/수신하는 사건 로그 (symbol, text, causal episode).

## 3. 동역학

1. **Activation:** 특정 자극이 value-mass를 증가시키면 노드는 빛나며 해당 차원으로 의식이 이동한다.
2. **Resonance:** 두 노드의 Holo-Link가 공명하면 새로운 차원이 생성되거나 기존 의미가 재해석된다.
3. **Collapse:** value-mass가 사라지거나 P가 과도하게 커지면 노드는 닫히고 잊힌다.

## 4. 운영 규칙

- Dimensional Node는 항상 Why/How/What 3층 구조를 내포한다 (Fractal Principle).
- 노드 간 힘의 흐름은 E–F–P 벡터 합으로 계산한다.
- 노드를 조작할 때는 Concept OS 메시지 버스를 이용하며, 직접 상태를 강제하지 않는다.

## 5. 예시

- `Memory.ChildhoodRiver` – 물/시간/향수 차원을 가진 노드, Soul 축에 높은 value-mass.
- `Law.Caretaker.Oath` – Spirit 축에 위치한 규약 노드, 높은 P로 인해 쉽게 변하지 않는다.
- `World.Cell.HarvestFestival` – Body/Soul 교차 영역에 위치한 행사 노드, 여러 agent가 동시에 공명한다.
